Menu Pembukuan digunakan Admin untuk mencatat jurnal keuangan instansi, melihat laporan keuangan (posisi keuangan dan laba rugi), mengatur bagi hasil, serta memigrasikan saldo dari sistem keuangan sebelumnya.

## Jurnal

Admin dapat menulis jurnal pembukuan keuangan. Langkah untuk akses jurnal pembukuan:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Jurnal**.
4. Sistem akan menampilkan halaman Jurnal.
5. Isi semua form jurnal.
6. Pilih **Kantor**, **Nama Jurnal**, **Akun Debit**, **Akun Kredit**, **Jumlah**, **Tanggal Entri**, **Catatan**, dan **Lampiran**.
7. Klik **Simpan** agar Admin dapat menyimpan data pembukuan.
8. Setelah berhasil disimpan, data akan tersimpan di riwayat jurnal.

## Riwayat Jurnal

Admin dapat melihat data jurnal pembukuan keuangan yang sudah pernah dibuat. Langkah untuk akses riwayat jurnal pembukuan:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Riwayat Jurnal**.
4. Sistem akan menampilkan halaman Riwayat Jurnal.
5. Halaman riwayat jurnal terdiri dari informasi kolom:
   - Tanggal
   - No. Referensi
   - Keterangan
   - Akun Debit
   - Akun Kredit
   - Nominal
   - Status
   - Kantor

## Laporan Posisi Keuangan

Admin dapat melihat data pembukuan laporan posisi keuangan pada setiap kantor. Langkah untuk akses laporan posisi keuangan:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Laporan Posisi Keuangan**.
4. Sistem akan menampilkan halaman Laporan Posisi Keuangan.
5. Sistem akan menampilkan posisi keuangan pada posisi aktiva dan pasiva.

## Laporan Laba Rugi

Admin dapat melihat data pembukuan laporan Laba Rugi pada setiap kantor. Langkah untuk akses laporan laba rugi:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Laporan Laba Rugi**.
4. Sistem akan menampilkan halaman Laporan Laba Rugi.
5. Sistem akan menampilkan laporan keuangan pada setiap kantor.

## List Pengaturan Bagi Hasil

Admin dapat melihat list data bagi hasil. Langkah untuk akses list bagi hasil:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Bagi Hasil**.
4. Kemudian, klik submenu **Pengaturan**.
5. Sistem akan menampilkan halaman List Bagi Hasil.

## Tambah Pengaturan Bagi Hasil

Admin dapat menambah data pengaturan bagi hasil. Langkah untuk tambah data bagi hasil:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Bagi Hasil**.
4. Kemudian, klik submenu **Pengaturan**.
5. Kemudian, klik tombol **Tambah**.
6. Sistem akan menampilkan halaman Tambah Bagi Hasil.
7. Admin mengisi semua form tambah bagi hasil yang terdiri dari:
   - Nama Penerima
   - Persentase
   - Kantor
   - Sumber

## Edit Pengaturan Bagi Hasil

Admin dapat mengubah data pengaturan bagi hasil. Langkah untuk mengubah data bagi hasil:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Bagi Hasil**.
4. Kemudian, klik submenu **Pengaturan**.
5. Kemudian, klik tombol **Edit**.
6. Sistem akan menampilkan halaman Edit Bagi Hasil.
7. Admin mengisi semua form edit bagi hasil yang terdiri dari:
   - Nama Penerima
   - Persentase
   - Kantor
   - Sumber

## Laporan Bagi Hasil

Admin dapat melihat laporan data bagi hasil. Langkah untuk akses laporan data bagi hasil:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Bagi Hasil**.
4. Kemudian, klik submenu **Laporan**.
5. Sistem akan menampilkan Laporan Bagi Hasil.

## Migrasi Akun

Admin dapat memindahkan saldo dari sistem keuangan sebelumnya ke sistem CARDS School. Langkah untuk akses migrasi data akun:

1. Login ke website, lalu klik menu **Keuangan**.
2. Kemudian, klik submenu **Pembukuan**.
3. Kemudian, klik submenu **Migrasi Akun**.
4. Sistem akan menampilkan form untuk Migrasi Akun.
5. Admin dapat mengisi semua form untuk migrasi akun.
