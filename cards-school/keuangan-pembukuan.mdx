Menu Pembukuan digunakan admin sekolah untuk mencatat jurnal keuangan instansi, melihat laporan keuangan (posisi keuangan dan laba rugi), mengatur bagi hasil, serta memigrasikan saldo dari sistem keuangan sebelumnya. Seluruh fitur diakses melalui menu **Keuangan** → submenu **Pembukuan**.

## Jurnal

Mencatat transaksi keuangan instansi sebagai entri jurnal pembukuan: pemasukan, pengeluaran, hutang-piutang, modal, hingga penyesuaian. Setiap entri memindahkan nilai dari satu akun ke akun lain dan menjadi dasar perhitungan laporan keuangan.

### Isian formulir

| Isian                   | Keterangan                                                      |
| ----------------------- | --------------------------------------------------------------- |
| Kantor                  | Kantor/instansi tempat jurnal dicatat.                          |
| Jenis Transaksi         | Menentukan label akun asal/tujuan (lihat tabel di bawah).       |
| Akun Asal & Akun Tujuan | Dipilih dari daftar akun; pilihan menyesuaikan jenis transaksi. |
| Jumlah                  | Nominal jurnal.                                                 |
| Tanggal Entri           | Default hari ini (zona waktu Asia/Jakarta).                     |
| Catatan                 | Keterangan tambahan.                                            |
| Lampiran                | Upload file bukti transaksi.                                    |

Jenis transaksi jurnal yang tersedia:

| Jenis Transaksi | Label akun asal → tujuan                    |
| --------------- | ------------------------------------------- |
| Pemasukan       | Diambil Dari → Simpan Ke                    |
| Pengeluaran     | Diambil Dari → Simpan Ke                    |
| Hutang          | Hutang Dari → Simpan Ke                     |
| Bayar Hutang    | Dibayar Menggunakan → Pembayaran Untuk      |
| Piutang         | Diberikan Dari → Untuk                      |
| Dibayar Piutang | Dibayar Dari → Simpan Ke                    |
| Tambah Modal    | Penambahan Untuk → Modal Berupa             |
| Tarik Modal     | Penarikan Berupa → Dari                     |
| Penyesuaian     | Saldo Yang Berkurang → Saldo Yang Bertambah |

<Callout kind="info" collapsed="false">
  Bagan akun sistem terbagi lima kategori: **Aset** (antara lain 101 Kas, 102 Tabungan Pada Bank, 103 Saldo CazhBox, 105 Piutang Usaha, 108 Aset Tetap & Inventaris, 111 Kas Tunai Tabungan), **Liabilitas** (201 Hutang Usaha, 202 Hutang Bank, 203 Hutang Pajak, 205 Simpanan Anggota), **Ekuitas** (301 Modal, 302 Hibah, 303 Prive, 304 Cadangan, 305 Saldo Laba/Rugi), **Pendapatan** (420 Pendapatan Produk/Jasa, 430 Pendapatan Lainnya, 440 Penerimaan Pembayaran Tagihan, 450 Pendapatan Pendaftaran Anggota, 460 Pendapatan Transaksi Tabungan), dan **Beban** (630 Tenaga Kerja, 640 Akomodasi & Transportasi, 650 Operasional, 660 Non Operasional, 670 Perlengkapan & ATK, 680 Sewa, 690 Penyusutan & Amortisasi, 611 Pajak, 612 Lain-lain).
</Callout>

## Riwayat Jurnal

Melihat data jurnal pembukuan yang sudah pernah dibuat.

### Informasi yang ditampilkan

Tabel Riwayat Jurnal menampilkan kolom: **Tanggal Entri**, **Instansi**, **Operator**, **Jenis Transaksi**, **Keterangan Transaksi**, **Nominal**, **Catatan**, dan **Lampiran**, diurutkan dari tanggal entri terbaru, default menampilkan bulan berjalan. Filter tersedia untuk **Instansi**, **Jenis Transaksi**, serta rentang tanggal **Awal Entri** dan **Akhir Entri**. Export Excel menghasilkan file **Riwayat Transaksi**; detail riwayat per akun dapat diekspor dengan nama file **Riwayat Transaksi (kode akun)**.

## Laporan Posisi Keuangan

Melihat laporan posisi keuangan pada setiap kantor.

<Callout kind="info" collapsed="false">
  Laporan difilter berdasarkan **Kantor**, **Tahun** (wajib), dan **Bulan** (wajib); default tahun dan bulan berjalan. Laporan terdiri dari bagian **Aset**, **Liabilitas**, **Ekuitas**, dan total **Liabilitas + Ekuitas**. Export Excel menghasilkan file **Laporan Posisi Keuangan.xlsx**.
</Callout>

## Laporan Laba Rugi

Melihat laporan laba rugi pada setiap kantor.

<Callout kind="info" collapsed="false">
  Laporan difilter berdasarkan **Kantor**, **Tahun**, dan **Bulan**, dan terdiri dari bagian **Pendapatan** dan **Beban** beserta totalnya. Export Excel menghasilkan file **Laporan Laba Rugi.xlsx**.
</Callout>

## List Pengaturan Bagi Hasil

Melihat daftar pengaturan bagi hasil yang telah dibuat.

## Tambah Pengaturan Bagi Hasil

Menentukan siapa penerima bagi hasil keuangan kantor beserta persentasenya. Pengaturan ini menjadi dasar perhitungan pada Laporan Bagi Hasil.

## Edit Pengaturan Bagi Hasil

Memperbarui penerima, persentase, kantor, atau sumber pada pengaturan bagi hasil yang sudah ada. Perhitungan pada Laporan Bagi Hasil mengikuti pengaturan ini.

## Laporan Bagi Hasil

Melihat laporan data bagi hasil.

### Informasi yang ditampilkan

Laporan Bagi Hasil difilter berdasarkan **Kantor**, **Tahun**, dan **Bulan**, dan menampilkan kolom: **Instansi**, **Penerima**, **% Bagi Hasil**, dan **Jumlah Bagi Hasil**, dihitung dari total pendapatan dan beban tiap kantor pada periode tersebut.

## Migrasi Akun

Memindahkan saldo dari sistem keuangan sebelumnya ke sistem CARDS School: mengisi saldo awal akun-akun pembukuan agar laporan keuangan melanjutkan angka dari sistem lama. Biasanya dilakukan sekali di awal penggunaan fitur pembukuan.

<Callout kind="info" collapsed="false">
  Migrasi akun hanya menampilkan akun internal sistem (akun eksternal seperti Saldo CazhBox dan Saldo Laba/Rugi dihitung otomatis), dan penyimpanan saldo awal memerlukan verifikasi **PIN**.
</Callout>

Migrasi akun memindahkan saldo dari sistem keuangan sebelumnya. Pastikan seluruh data yang diisi sudah benar sebelum menjalankan migrasi.
