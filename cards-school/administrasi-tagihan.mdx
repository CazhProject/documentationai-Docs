Menu Tagihan digunakan admin sekolah untuk membuat dan memonitor seluruh tagihan anggota (siswa, guru, dan staf), memantau pembayaran, serta menindaklanjuti tunggakan.

Untuk pembayaran tagihan oleh Admin, lihat [Keuangan: Transaksi](/cards-school/keuangan-transaksi). Orang tua juga dapat membayar tagihan langsung dari aplikasi; lihat panduan [CARDS Parent](/cards-parent/keuangan).

## Daftar Data Tagihan

Memonitor data tagihan seluruh anggota.

### Informasi yang ditampilkan

Tabel Data Tagihan menampilkan kolom: **Status**, **Nama**, **Ref ID**, **Jenis Tagihan**, **Catatan**, **Jumlah Tagihan**, **Terbayar**, dan **Sisa Tagihan** (kolom Nomor Kartu tersembunyi secara default). Data dapat difilter berdasarkan **Instansi**, **Jenis Tagihan**, **Tingkat**, **Kelas**, **Status**, serta rentang **Dari Tanggal** dan **Sampai Tanggal**; jumlah baris per halaman dapat dipilih 10/50/100/500/1000/2000. Status tagihan terdiri dari:

| Status                  | Keterangan                                    |
| ----------------------- | --------------------------------------------- |
| **Lunas**               | Tagihan telah dibayar penuh.                  |
| **Terbayar Sebagian**   | Tagihan baru dibayar sebagian.                |
| **Belum Terbayar**      | Belum ada pembayaran.                         |
| **Menunggu Pembayaran** | Invoice sudah dibuat dan menunggu pembayaran. |

## Edit Data Tagihan

Memperbarui data tagihan anggota, misalnya mengoreksi nominal tagihan yang salah dibuat atau melengkapi catatan. Perubahan nominal memengaruhi jumlah yang harus dibayar anggota, jadi pastikan nilainya benar sebelum menyimpan.

<Callout kind="info" collapsed="false">
  Aturan sistem: jika tagihan berstatus **Belum Terbayar**, hanya nominal tagihan yang dapat diubah. Untuk tagihan yang sudah memiliki pembayaran, perubahan dilakukan melalui alur penyesuaian (adjustment) per pembayaran beserta catatannya.
</Callout>

## Hapus Data Tagihan

Membatalkan tagihan yang belum terbayar, misalnya tagihan yang salah dibuat atau tidak jadi ditagihkan, secara satuan maupun massal. Tagihan yang dihapus tidak lagi tampil di daftar tagihan anggota dan tidak lagi dapat dibayar.

Hanya tagihan berstatus belum terbayar yang dapat dihapus, dan penghapusan bersifat permanen.

## Buat Tagihan Satuan

Membuat tagihan baru untuk satu anggota tertentu. Tagihan yang dibuat tampil di Data Tagihan dan pada tab Tagihan anggota bersangkutan, serta dapat dibayar melalui menu Keuangan → Transaksi atau aplikasi orang tua.

### Isian formulir

| Isian                      | Ketentuan                           |
| -------------------------- | ----------------------------------- |
| Instansi                   | Wajib.                              |
| Anggota (Nama/Nomor Kartu) | Wajib; dipilih dari daftar anggota. |
| Jenis Tagihan              | Wajib.                              |
| Jumlah Tagihan             | Wajib, angka.                       |
| Catatan                    | Opsional.                           |

## Buat Tagihan Massal

Membuat tagihan yang sama untuk banyak anggota sekaligus berdasarkan jenis anggota atau tag; praktis untuk tagihan rutin seperti SPP. Setiap anggota terpilih menerima tagihannya masing-masing.

<Callout kind="info" collapsed="false">
  Pada mode massal, penerima dapat dipersempit: untuk **Siswa** berdasarkan Tingkat, Kelas, dan Jurusan; untuk **Guru** berdasarkan jenis guru; untuk **Staf** berdasarkan jenis staf; serta berdasarkan **Tag** (dapat memilih lebih dari satu).
</Callout>

## Import Data Tagihan

Membuat tagihan melalui import file; cocok ketika nominal tagihan berbeda-beda per anggota, karena nominal diisi per baris pada file template.

<Callout kind="info" collapsed="false">
  Aturan import: file wajib berformat **.xls** dengan ukuran maksimal **4 MB (4096 KB)**. Template yang diunduh sudah terisi anggota **aktif** sesuai filter (instansi, jenis anggota, tag). Kolom template: **Nama** (opsional), **Nomor Kartu** (wajib, divalidasi terhadap instansi), **Jumlah** (wajib, angka, minimal 1), dan **Catatan** (opsional). Untuk jenis anggota Guru/Staf, jenis guru/staf wajib dipilih sebelum import; setiap import tercatat pada halaman **Log Import**, dan anggota menerima notifikasi tagihan baru.
</Callout>

## Laporan Pembayaran Tagihan

Memonitor laporan pembayaran tagihan seluruh anggota.

## Daftar Rekap Pembayaran Tagihan

Memonitor rekap pembayaran setiap jenis tagihan.

### Informasi yang ditampilkan

Rekap dikelompokkan per **jenis tagihan** dan **instansi**, menampilkan total pembayaran diterima, total tagihan, dan **persentase penerimaan**. Filter tersedia untuk instansi, operator, dan rentang tanggal (default: bulan berjalan). Export Excel menghasilkan file **Rekap Pembayaran.xlsx**.

## Detail Rekap Pembayaran Tagihan

Melihat detail rekap pembayaran untuk satu jenis tagihan.

## Tunggakan Tagihan

Memonitor tunggakan tagihan seluruh anggota.

## Detail Tunggakan Tagihan

Menindaklanjuti tunggakan per anggota.\
\
Pengingat Tagihan
-----------------

**Edit Pengingat Tagihan&#x20;**&#x6D;emungkinkan admin mengubah pengaturan pengingat tagihan yang telah dibuat sesuai kebutuhan. Langkah-langkah sebagai berikut :

1. Login ke website, kemudian klik menu Pengaturan.
2. Kemudian klik submenu Tagihan.
3. Klik/Pilih submenu Pengingat Tagihan.
4. Menampilkan Halaman Pengingat Tagihan.
5. Sistem menampilkan list data Pengingat Tagihan.
6. Klik button action Edit pada salah satu data.

Ubah data sesuai yang diinginkan, lalu klik button Simpan.\
\
Legalitas Bukti Pembayaran
--------------------------

**Legalitas Bukti Pembayaran** memungkinkan admin mengatur informasi legalitas yang akan ditampilkan pada bukti pembayaran sebagai informasi resmi transaksi. Langkah-langkah sebagai berikut :

1. Login ke website, kemudian klik menu Pengaturan.
2. Kemudian klik submenu Tagihan.
3. Klik/Pilih submenu Legalitas Bukti Pembayaran.
4. Menampilkan popup Legalitas Bukti Bayar.
5. Input data Legalitas Bukti Bayar yang diinginkan, lalu klik button Simpan.
