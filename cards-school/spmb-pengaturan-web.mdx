Menu Pengaturan Web digunakan admin sekolah untuk mengelola tampilan dan konfigurasi website [SPMB](/referensi/glosarium#spmb) (Sistem Penerimaan Murid Baru) lembaga. Seluruh pengaturan diakses melalui menu **Anggota** → submenu **SPMB** → menu **Pengaturan Web**, lalu pilih tab yang sesuai. Pengaturan berlaku per instansi (pilih instansi terlebih dahulu).

## Profil

Mengedit profil yang tampil pada landing page SPMB.

### Isian formulir Profil

| Field                           | Wajib | Keterangan                                                         |
| ------------------------------- | ----- | ------------------------------------------------------------------ |
| Judul Web                       | Ya    | Judul yang tampil pada landing page SPMB.                          |
| Status SPMB                     | Ya    | Aktif / Tidak Aktif; mengaktifkan atau menonaktifkan website SPMB. |
| Nama PIC                        | Tidak | Penanggung jawab SPMB.                                             |
| No HP PIC                       | Tidak | -                                                                  |
| Tahun Ajaran SPMB               | Tidak | Tahun ajaran tujuan pendaftaran.                                   |
| Nomor Urut Pendaftaran          | Tidak | Awal penomoran nomor pendaftar.                                    |
| Notifikasi WhatsApp             | Tidak | Aktif/nonaktif pengiriman notifikasi WhatsApp.                     |
| Telepon, Alamat, Email instansi | Tidak | Kontak yang tampil di landing page.                                |
| Logo                            | Tidak | jpg/png/jpeg/heic/heif, maksimal 10 MB.                            |

Halaman ini juga menampilkan **URL website SPMB** lembaga (berdasarkan slug instansi).

## Beranda

Mengedit tampilan alur pendaftaran pada landing page SPMB.

Daftar tahapan alur pendaftaran ditampilkan dalam tabel dengan kolom: **Urutan**, **Nama Tahapan**, **Deskripsi**, **Ilustrasi**, dan **Tampilkan** (toggle). Setiap tahapan dapat diedit dengan isian: deskripsi (wajib, maksimal 255 karakter), jenis ilustrasi (**gambar** jpg/jpeg/png maksimal 2 MB, atau **animasi Lottie** via URL), serta visibilitas.

Tahapan alur otomatis disinkronkan dengan daftar status pendaftaran (termasuk status kustom) dan posisi pembayaran pada jadwal (Setelah Pendaftaran / Setelah Pengumuman).

## Jadwal

Mengelola gelombang (jadwal) pendaftaran SPMB beserta kuota dan biayanya. Jadwal menentukan kapan calon pendaftar dapat mendaftar, berapa kuota penerimaannya, dan berapa biaya yang ditagihkan.

Jadwal yang dihapus tidak lagi tampil di daftar dan tidak lagi tersedia sebagai gelombang pendaftaran. Berhati-hati menghapus jadwal yang sudah memiliki pendaftar.

Daftar jadwal menampilkan kolom: **Nama Jadwal**, **Satu Harga**, **Tanggal**, **Kuota**, dan **Status**.

### Isian formulir Jadwal

| Field                   | Wajib | Keterangan                                                                           |
| ----------------------- | ----- | ------------------------------------------------------------------------------------ |
| Nama Jadwal             | Ya    | -                                                                                    |
| Tanggal Mulai & Selesai | Ya    | Periode gelombang pendaftaran.                                                       |
| Jenis Kuota             | Ya    | **Global** (satu kuota untuk semua) atau **Per Jalur Penerimaan** (kuota per jalur). |
| Kuota                   | Tidak | Jumlah kuota penerimaan (global atau per jalur).                                     |
| Satu Harga              | Tidak | Jika aktif, satu harga untuk semua jurusan; jika tidak, harga diatur per jurusan.    |
| Jenis Tagihan           | Tidak | Jenis invoice untuk biaya pendaftaran.                                               |
| Harga                   | Tidak | Biaya SPMB.                                                                          |
| Harga Coret             | Tidak | Harga sebelum diskon (tampilan).                                                     |
| Minimum Pembayaran      | Tidak | Minimum bayar (cicilan); pada lembaga non-finansial otomatis sama dengan harga.      |
| Posisi Pembayaran       | Ya    | **Setelah Pendaftaran** atau **Setelah Pengumuman**.                                 |

## Status Kustom

Mengelola status kustom: tahapan seleksi tambahan buatan lembaga (misalnya "Tes Wawancara") yang melengkapi status bawaan pendaftaran. Status kustom yang aktif tampil dalam alur pendaftaran di landing page dan dapat dipilih saat mengubah status pendaftar.

Berhati-hati menghapus status kustom yang masih memiliki pendaftar (lihat kolom **Total Pendaftar**). Status yang dihapus tidak lagi tersedia dalam alur pendaftaran.

Daftar status kustom menampilkan kolom: **Instansi**, **Nama Status**, **Urutan**, **Status**, **Total Pendaftar**, dan **Dibuat Pada**. Isian saat menambah/mengedit: nama status, urutan (menentukan posisi dalam alur), dan status aktif.

## Formulir Standar

Mengatur field bawaan sistem yang ditampilkan pada formulir pendaftaran calon peserta didik: menentukan field mana yang tampil, wajib diisi, dan berlaku untuk jalur penerimaan mana.

Formulir standar terdiri dari ±54 field bawaan sistem yang terbagi dalam 5 kategori:

| Kategori                  | Contoh field                                                                                                                                                                                                                                                                                                                                                          |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data Pendidikan**       | Jurusan, Jalur Penerimaan, Sistem Pendidikan.                                                                                                                                                                                                                                                                                                                         |
| **Data Diri Calon Siswa** | Email, NIK/KIA, NISN, NPSN, Asal Sekolah, Masa belajar pendidikan sebelumnya, Tanggal dan Nomor Ijazah, KIP, Pas Foto 3x4, Saudara kandung yang sudah sekolah disini, Status dalam keluarga, Jumlah saudara, Moda transportasi, Jarak menuju sekolah (KM), Waktu tempuh, Nama panggilan, Kewarganegaraan, Agama, Status orang tua, Bahasa keseharian, Status tinggal. |
| **Data Kesehatan**        | Riwayat kesehatan, Disabilitas, Tinggi badan, Berat badan, Tes buta warna, Golongan darah.                                                                                                                                                                                                                                                                            |
| **Data Orang Tua**        | Nama, NIK, Pendidikan, Pekerjaan, Status pernikahan, Nomor HP, Tempat & tanggal lahir, Agama, Kewarganegaraan (untuk Ayah dan Ibu), Penghasilan orang tua.                                                                                                                                                                                                            |
| **Data Dokumen/File**     | Kartu Keluarga, Ijazah/Keterangan Lulus, File lainnya.                                                                                                                                                                                                                                                                                                                |

Setiap field memiliki toggle **Ditampilkan** dan **Wajib diisi** (beberapa field inti tidak dapat diubah), serta dapat dibatasi hanya tampil pada jalur penerimaan tertentu.

## Formulir Kustom

Mengelola field tambahan buatan lembaga yang ditampilkan pada formulir pendaftaran calon peserta didik, untuk data yang tidak tercakup formulir standar.

Daftar formulir kustom menampilkan kolom: **Label**, **Kategori**, **Jenis Input**, **Ditampilkan?**, **Diwajibkan?**, dan **Jalur Penerimaan**.

### Isian formulir Kustom

| Field            | Wajib | Keterangan                                                                                          |
| ---------------- | ----- | --------------------------------------------------------------------------------------------------- |
| Label            | Ya    | Nama field yang tampil pada formulir.                                                               |
| Deskripsi        | Tidak | Keterangan tambahan.                                                                                |
| Kategori         | Ya    | Data Diri Calon Siswa / Data Kesehatan / Data Orang Tua / Data Dokumen (File).                      |
| Jenis Input      | Ya    | Teks, Tanggal, Textarea (teks panjang), atau Dropdown; kategori Dokumen otomatis berjenis **File**. |
| Opsi             | -     | Daftar pilihan, hanya untuk jenis Dropdown.                                                         |
| Wajib diisi      | Tidak | Default: ya.                                                                                        |
| Ditampilkan      | Tidak | Default: ya.                                                                                        |
| Jalur Penerimaan | Tidak | Batasi field untuk jalur tertentu.                                                                  |

## Program Khusus

Mengelola jalur penerimaan dan sistem pendidikan yang menjadi pilihan calon pendaftar pada formulir pendaftaran. Jalur penerimaan dapat disertai potongan biaya pendaftaran; sistem pendidikan mencatat program belajar yang tersedia.

| Opsi                  | Kemampuan                                                                           |
| --------------------- | ----------------------------------------------------------------------------------- |
| **Jalur Penerimaan**  | Melihat list jalur penerimaan, edit, hapus, dan tambah jalur penerimaan.            |
| **Sistem Pendidikan** | Melihat daftar sistem pendidikan, edit, hapus, dan tambah daftar sistem pendidikan. |

Isian jalur penerimaan: **Nama** jalur, **Tipe potongan** (Nominal atau Persentase), dan **Nominal potongan**, yaitu potongan biaya pendaftaran untuk pendaftar jalur tersebut. Isian sistem pendidikan: nama program saja.

## Daftar Ulang

Mengelola [tagihan](/referensi/glosarium#tagihan) daftar ulang: rincian biaya per jurusan yang otomatis dibebankan kepada siswa yang dinyatakan Diterima, baik siswa baru maupun siswa lama.

Tersedia dua daftar tagihan: **daftar ulang siswa baru** dan **daftar ulang siswa lama**, keduanya dengan kolom: **Nama Jurusan**, **Jenis**, **Jumlah**, dan **Catatan**. Isian tagihan: jurusan, jenis tagihan (invoice type), jumlah nominal, dan catatan.

Tagihan akan ditambahkan otomatis kepada siswa yang dinyatakan **Diterima**. Tab Daftar Ulang hanya tampil jika fitur keuangan lembaga aktif.

## Kunjungi Website

Admin memiliki kemampuan untuk akses website secara langsung,

Langkah untuk mengunjungi website:

1. Login ke website, kemudian klik menu **Anggota.**
2. System akan menampilkan submenu **SPMB.**
3. Pilih menu **pengaturan web.**
4. Pilih menu submenu **lainnya.**
5. Pilih menu **kunjungi website.**
6. System akan **redirect&#x20;**&#x6B;e landingpage website SPMB.

## Analytic

Memonitor jumlah pengunjung website SPMB. Admin memiliki kemampuan untuk monitoring analytic yang terjadi pada halaman website SPMB. Langkah untuk monitoring analytic:

1. Login ke website, kemudian klik menu **Anggota.**
2. System akan menampilkan submenu **SPMB.**
3. Pilih menu **pengaturan web.**
4. Pilih menu submenu **lainnya.**
5. Pilih menu **analytic.**
6. Pada halaman analytic, Admin dapat melihat jumlah pengunjung website.
