Modul Siswa adalah pusat pengelolaan data siswa bagi admin sekolah: mulai dari menambah dan mengubah data, memindahkan siswa antar kelas atau tahun ajaran, hingga menjadikan siswa sebagai alumni. Seluruh fitur diakses melalui menu **Anggota** → submenu **Siswa**, yang menampilkan **Halaman Siswa**.

Pada halaman Siswa, admin dapat mengelola:

- Tambah data siswa secara satuan dan massal.
- Edit detail data siswa.
- Mengubah siswa menjadi Alumni.
- Export data siswa menjadi file Excel.

## Informasi yang ditampilkan

Tabel daftar siswa menampilkan kolom: **Nama**, **Nomor Kartu**, **Kelas**, **Tahun Ajaran**, **Instansi**, **Kamar**, **Status**, **Nama Ayah**, **Nama Ibu**, **Status Pengkinian Data**, dan **Terakhir Diubah**. Pencarian (search) mencakup seluruh kolom tersebut plus **NISN**.

Filter yang tersedia: **Instansi**, **Tahun Ajaran**, **Jurusan**, **Tingkat**, **Kelas**, **Kamar**, dan **Status Pengkinian Data** (Sudah dilakukan / Belum dilakukan). Jumlah baris per halaman dapat dipilih: 10, 50, 100, 500, 1000, atau 2000.

**Export** menghasilkan file Excel (`Siswa.xlsx`) berisi baris terpilih (checkbox) atau seluruh halaman aktif, dengan ±65 kolom lengkap: identitas (ID, Nomor Kartu, Nomor Induk, Kode QR, Nama Lengkap, NIK, NISN, NPSN, KIP), penempatan (Instansi, Tahun Ajaran, Jurusan, Program, Kamar, Sistem Pendidikan, Kelas, Status), data diri (Jenis Kelamin, Tempat/Tanggal Lahir, Alamat, Email, No Telp, Agama, Kewarganegaraan, Status Keluarga/Tinggal, transportasi, jarak & waktu tempuh sekolah), data kesehatan (Riwayat Kesehatan, Golongan Darah, Tinggi/Berat Badan, Disabilitas), serta blok lengkap data Ayah dan Ibu (Nama, NIK, Pendidikan, Pekerjaan, Status Pernikahan, No Telp, Kewarganegaraan, Agama, Tempat/Tanggal Lahir) dan Penghasilan Orang Tua.

Nilai status keanggotaan yang dikenal sistem: **ACTIVE** (Aktif), **INACTIVE** (Tidak Aktif), dan **GRADUATED** (Lulus). Aksi Ubah Status pada halaman Siswa hanya menerima ACTIVE atau INACTIVE.

## Pindah Data Siswa

Opsi **Pindah Data Siswa** memuat sejumlah aksi massal untuk memindahkan atau memperbarui data siswa terpilih, misalnya menaikkan kelas, memindahkan tahun ajaran, atau mengubah status. Aksi ini mengubah data penempatan siswa yang dipilih saja; data diri siswa lainnya tidak berubah. Alur dasarnya sama untuk semua aksi.

| Aksi                         | Data yang diisi                                        |
| ---------------------------- | ------------------------------------------------------ |
| **Pindah Tahun Ajaran**      | Data tahun ajaran tujuan.                              |
| **Pindah Tingkat**           | Data tingkat tujuan.                                   |
| **Pindah Kelas**             | Data kelas tujuan.                                     |
| **Pindah Jurusan**           | Data jurusan tujuan.                                   |
| **Pindah Alumni**            | Tidak ada isian tambahan, langsung klik **Simpan**.    |
| **Pindah Instansi**          | Data instansi tujuan.                                  |
| **Ubah Status**              | Isi atau pilih data status baru (Aktif / Tidak Aktif). |
| **Tambah Tag Siswa**         | Pilih data tag yang ingin ditambahkan pada siswa.      |
| **Pindah Kamar**             | Data kamar tujuan.                                     |
| **Pindah Jalur Penerimaan**  | Jalur penerimaan (program SPMB) tujuan.                |
| **Pindah Sistem Pendidikan** | Sistem pendidikan tujuan.                              |

Siswa yang dipindah ke alumni akan berpindah pengelolaannya ke menu [Anggota: Alumni](/cards-school/anggota-alumni) dan dapat diaktifkan kembali dari sana.

## Tambah dan Import Siswa

Mendaftarkan siswa baru ke instansi: satuan melalui formulir, atau massal melalui import file Excel. Siswa yang berhasil ditambahkan langsung tampil di daftar Siswa dan dapat dikelola seperti anggota lain.

### Isian formulir Tambah Siswa

**Data Diri:**

| Field                         | Wajib | Keterangan                                     |
| ----------------------------- | ----- | ---------------------------------------------- |
| Instansi                      | Ya    | Instansi tempat siswa terdaftar.               |
| Nama Lengkap                  | Ya    | -                                              |
| Jenis Kelamin                 | Ya    | Laki - laki / Perempuan.                       |
| Email                         | Ya    | Harus unik antar anggota.                      |
| No HP                         | Ya    | Angka, divalidasi format nomor telepon.        |
| No Induk                      | Ya    | Nomor induk anggota.                           |
| Nomor Kartu                   | Ya    | Angka; dapat divalidasi dengan tombol **Cek**. |
| Tahun Ajaran                  | Ya    | -                                              |
| Tingkat                       | Ya    | -                                              |
| Kelas                         | Ya    | -                                              |
| NISN                          | Tidak | -                                              |
| Tempat Lahir, Tanggal Lahir   | Tidak | Format tanggal Y-m-d.                          |
| Alamat Lengkap                | Tidak | -                                              |
| Asal Sekolah, NPSN, Nomor KIP | Tidak | NPSN dan KIP maksimal 20 karakter.             |
| No Identitas (NIK)            | Tidak | Angka 16 digit.                                |
| Jurusan, Kamar                | Tidak | -                                              |
| UID Kartu                     | Tidak | Harus unik.                                    |
| Tag                           | Tidak | Pilihan ganda dari tag di instansi.            |
| Foto                          | Tidak | jpg/png/jpeg/heic/heif, maksimal 10 MB.        |

**Data Orang Tua** (semua opsional): Nama Ayah, NIK Ayah, Pekerjaan Ayah (pilihan dari daftar pekerjaan), No HP Ayah, serta field yang sama untuk Ibu.

### Kolom template import

Import siswa menggunakan template Excel (`template-member-v9.xls`, format .xls maksimal 4 MB; baris judul di baris ke-3, data mulai baris ke-4) dengan kolom: `nama_lengkap` (wajib), `tempat_lahir`, `tanggal_lahir` (d/m/Y), `jenis_kelamin` (wajib: L/P), `alamat_lengkap`, `email` (wajib, unik), `nomor_handphone` (angka 11–15 digit), `nomor_induk_member` (wajib), `nik` (16 digit), `nomor_kartu` (wajib, angka), `uid_kartu` (unik), `tag` (dipisah koma, kode tag harus terdaftar), `rfid`, `nisn`, `nama_ayah`, `nama_ibu`, dan `kamar` (harus cocok nama kamar aktif di instansi). Selain import data anggota, tersedia jenis import **Foto Anggota** (foto jpeg/png/jpg/heic/heif maksimal 2 MB per file).

Halaman **Log Import** menampilkan kolom: Tanggal Import, Instansi, Tahun Ajaran, Tingkat, Kelas, Jurusan, Jenis Import, Total Data, dan Status (PENDING / SUCCESS / FAILED), dengan filter rentang tanggal dan penempatan.

## Detail Siswa

Setiap siswa memiliki halaman detail dengan sejumlah tab. Untuk mengaksesnya: menu **Anggota** → submenu **Siswa** → pilih **Detail** pada data siswa yang diinginkan, lalu pilih tab yang diperlukan pada navbar detail profil siswa.

<Image src="https://blob-cdn.documentation.ai/org-9f0069f2-e3f7-48a0-be6a-ee3f46a1d606/doc-0793c7e9-51db-4bf5-85cf-91406e00f125/1784256721914-4i8cqswz9j4-pasted-image-1784256718175.png?fm=auto&auto=compress%2Cformat" width="2560" height="1280" alt="" />

Tab yang tampil untuk siswa: **Data Diri**, **Data Orang Tua**, **Kartu**, **Tagihan**, **Dokumen**, **Rapor**, **Kesehatan**, **Pelanggaran**, **Prestasi**, **Progres**, **Perizinan**, serta di submenu Lainnya: **Tahfidz**, **Kunjungan**, **Perundungan**, dan **Tugas** (tab Tugas hanya muncul jika fitur tugas diaktifkan pada pengaturan lembaga).

<Image src="https://blob-cdn.documentation.ai/org-9f0069f2-e3f7-48a0-be6a-ee3f46a1d606/doc-0793c7e9-51db-4bf5-85cf-91406e00f125/1784267598817-jprpkrkfmoq-pasted-image-1784267594845.png?fm=auto&auto=compress%2Cformat" width="2560" height="1258" alt="" />

Berhati-hati dengan aksi **Hapus** pada tab-tab detail siswa (dokumen, riwayat kesehatan, pelanggaran, prestasi, perizinan, progress, kunjungan, perundungan, tahfidz). Data yang dihapus tidak lagi tampil di daftar dan tidak ada tombol untuk mengembalikannya dari halaman yang sama.
